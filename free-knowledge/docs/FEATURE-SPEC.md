# free-civics — Complete Feature Specification

**Project:** free-civics: Know Your Government
**Platform:** Next.js web application (React + SSR)
**Purpose:** A free public service providing comprehensive, transparent information about U.S. government officials to support informed civic participation
**Driving Principle:** Transparency and informed decision-making — present data, not opinions. Users draw their own conclusions.

---

## I. CORE SEARCH & NAVIGATION

### 1.1 Official Search
- Search bar as the primary landing page element (Google-style)
- Search by name, title, or position (e.g., "Nancy Pelosi", "Speaker of the House", "Senator from Texas")
- Auto-suggest/typeahead as users type
- Search results page when multiple matches are found
- Support for current and former officials

### 1.2 Zip Code Lookup ("My Representatives")
- Enter zip code to see all government officials representing that location
- Returns: U.S. Representative, two U.S. Senators, President, relevant Cabinet heads
- Display official name, title, party, contact info, and link to full profile
- Congressional district identification via Census geocoder
- Quick-action links (phone, website, full profile)

### 1.3 Position-Based Browsing
- Browse all members of Congress (House and Senate)
- Browse by state or party
- Browse President and Cabinet heads
- Browse by committee membership

## II. OFFICIAL PROFILE — OVERVIEW

### 2.1 Profile Header
- Official portrait/photo
- Full name and title
- Party affiliation badge
- State and district (if applicable)
- Chamber (House / Senate / Executive)
- Years serving / term start date
- Quick stats bar (bills sponsored, bills enacted, total raised, small donor %, committee count)

### 2.2 AI-Synthesized Summary
- 2-3 paragraph overview of the official's public service career
- Written in neutral, encyclopedic tone
- Sourced from Wikipedia, Wikidata, Congress.gov, and news
- Non-partisan — factual only

### 2.3 Biography
- Organized chronologically: Early Life, Education, Path to Office, Congressional Career
- Professional background before government
- Educational history with institutions, degrees, fields
- Military service (if applicable)
- Source attribution for all claims

### 2.4 Key Facts Panel
- Structured quick-reference data: party, state, district, birth date, education, years in office
- Formatted as label/value pairs in a sidebar card

## III. OFFICIAL PROFILE — LEGISLATION

### 3.1 Sponsored Bills
- List of all bills this official has introduced as primary sponsor
- Each bill shows: bill ID, title, AI plain-language summary, current status, date introduced, latest action
- Status badges: Introduced, In Committee, Passed Chamber, Passed Both, Signed Into Law, Vetoed, Failed
- Link to full bill text on Congress.gov
- Total count of sponsored bills and bills enacted into law

### 3.2 Co-Sponsored Bills
- Bills this official has co-sponsored (supporting another member's legislation)
- Same display format as sponsored bills
- Total count

### 3.3 AI Bill Summaries
- Every bill gets a plain-language summary generated by Claude
- Translates legislative jargon into clear language any voter can understand
- Batch processing for efficiency (5 bills per API call)

### 3.4 Detailed Bill Analysis (Deep Dive)
- For individual bills: key provisions, potential impact, controversial aspects, related issues
- Available on click/expand for any bill in the list

### 3.5 Voting Record
- Complete record of how this official voted on floor votes
- Each vote shows: date, question/description, member's position (YEA/NAY/NOT VOTING), overall result, party breakdown
- AI-generated plain-language description of what was voted on
- Vote categorized by issue area (Healthcare, Economy, Education, etc.)
- Filterable by issue category, date range, and vote position

### 3.6 Historical Voting Record Timeline
- Visual timeline of votes over the official's career
- Filterable by issue area
- Shows patterns and trends over time

## IV. OFFICIAL PROFILE — FINANCES

### 4.1 Campaign Finance Overview
- Current election cycle totals: raised, spent, cash on hand
- Funding breakdown: from individuals, from PACs, self-financed
- Small donor percentage (donations under $200)
- Source attribution to FEC

### 4.2 Top Donors
- List of top contributing organizations/individuals
- Shows: name, type (individual/organization/PAC), total amount, industry
- Ranked by contribution amount

### 4.3 Top Contributing Industries
- Industries ranked by total contributions
- Breakdown: from individuals vs. from PACs within each industry
- Ranked list with amounts

### 4.4 PAC Contributions
- Super PACs, traditional PACs, and leadership PACs supporting this official
- PAC name, total amount, type, and ideological leaning

### 4.5 Money Trail (Donor-to-Vote Correlation)
- Maps industry contributions to related voting patterns
- For each top contributing industry: shows relevant votes and whether the official's position aligned with industry interests
- Correlation score presented as raw data — users assess significance themselves
- Non-editorialized: presents facts, not conclusions

### 4.6 Personal Financial Disclosures
- Estimated net worth range (from disclosure filings)
- Assets, liabilities, and transactions
- Link to official disclosure documents
- Disclosure year noted

### 4.7 Previous Election Cycles
- Historical campaign finance data from prior cycles
- Trend visualization of fundraising over time

## V. OFFICIAL PROFILE — METRICS DASHBOARD (formerly Scorecard)

**Design Principle:** Present raw performance metrics with context and benchmarks. NO letter grades, NO scores, NO editorial ratings. Users assess their officials themselves.

### 5.1 Attendance & Participation Metrics
- Vote attendance rate (% of votes participated in)
- Total votes cast vs. total votes missed
- Benchmark: chamber average attendance rate for comparison
- Trend indicator (improving/declining/stable)

### 5.2 Legislative Activity Metrics
- Total bills sponsored and co-sponsored
- Bills that advanced past committee
- Bills passed in chamber
- Bills enacted into law
- Benchmark: chamber average for each metric
- Legislative effectiveness rank within chamber (if available)

### 5.3 Bipartisanship Metrics
- Cross-party vote percentage (how often they vote against their own party majority)
- Total cross-party votes count
- Benchmark: chamber and party averages
- Most notable cross-party votes listed

### 5.4 Transparency Metrics
- Small donor funding percentage
- Financial disclosure filing status (up to date or not)
- Benchmark: chamber average small donor percentage

### 5.5 Constituency Focus Metrics
- Percentage of sponsored/co-sponsored bills focused on state or local issues
- Count of locally-focused legislation
- Benchmark: chamber average

### 5.6 Leadership & Committee Metrics
- Number of committee assignments
- Leadership positions held (chair, ranking member, vice chair)
- List of specific committees and roles
- Benchmark: typical committee assignment count

### 5.7 Third-Party Ratings (Raw Data)
- Ratings from external organizations (NRA, ACLU, Sierra Club, Heritage Foundation, etc.)
- Presented as raw data: organization name, rating/score, year, category
- No aggregation or weighting — users decide which ratings matter to them

### 5.8 Benchmark Comparisons
- All metrics shown alongside: chamber average, party average, state delegation average
- Presented as data tables or bar charts — no editorial interpretation

## XIII. TECHNICAL ARCHITECTURE

### 13.1 Core Engine (free-knowledge)
- Adapter pattern: each data source is an isolated module implementing a common interface
- Pipeline architecture: Collection → Normalization → Synthesis → Presentation
- Configuration-driven verticals: new versions are config profiles, not separate codebases

### 13.2 Government Adapters
- Congress Adapter (Congress.gov API)
- Campaign Finance Adapter (FEC API)
- Civic Info Adapter (Google Civic + Census)
- Wikipedia Adapter
- Wikidata Adapter
- News Adapter

### 13.3 Specialized Engines
- Legislation Engine (AI bill summaries, vote categorization)
- Metrics Engine (raw metric calculation with benchmarks — no grading)
- Compare Engine (side-by-side analysis)
- Issues Engine ("What Affects Me" filtering and reporting)

### 13.4 Caching
- In-memory LRU cache for development
- Redis for production
- Adapter-level caching (30 min TTL)
- Profile-level caching (30 min TTL)
- Zip lookup caching (24 hour TTL)

### 13.5 Tech Stack
- Framework: Next.js 14+ (App Router)
- Language: TypeScript
- Frontend: React with custom component library
- Fonts: Playfair Display + Source Sans 3
- Color System: Civil rights era inspired — mahogany, gold, warm ivory
- Hosting: Vercel (free tier to start)
- Database: PostgreSQL (for user accounts when premium features launch)
